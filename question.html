The Difference Between Map,Filter,forEach and Find?

ANS :

map, filter, forEach, and find are all array methods in JavaScript that operate on arrays and return a new array or a single value. The main differences between them are:

1.map: returns a new array with the same length as the original array, where each element is the result of applying a callback function to the corresponding element in the original array.

2.filter: returns a new array containing all elements of the original array that satisfy a certain condition defined by a callback function.

3.forEach: calls a callback function for each element in the array, but does not return a new array.

4.find: returns the first element in the array that satisfies a certain condition defined by a callback function, or undefined if no such element is found.

What Is The Difference Between Local Storage , Session Storage And Cookies?

ANS :

1. Local Storage: Local storage is a client-side storage mechanism that allows web applications to store data on a user's device without any expiration date. The data is stored as key-value pairs, and can only be accessed by scripts on the same domain. Local storage has a much larger storage capacity than cookies, with most browsers supporting up to 10MB of data.

2.Session Storage: Session storage is similar to local storage, but the data is cleared once the user closes the browser window. It is useful for storing data that needs to be accessed during a user's session on a website, but does not need to persist beyond that session. Like local storage, session storage has a larger storage capacity than cookies, with most browsers supporting up to 10MB of data. In summary, cookies are used to store small amounts of data with a defined expiration date, while local storage and session storage provide larger storage capacity and do not have expiration dates (or have one tied to a session). Local storage is meant for long-term storage while session storage is useful for short-term storage within a session.

3. Cookies: Cookies are small text files that are created by a website and stored on the user's device. They are used to store user preferences, login information, and other data that can be accessed by both the client-side and server-side scripts. Cookies have a lifespan, which is determined by the expiration date set by the website. They can be set to either be persistent (which means that they will remain on the user's device even after they close the browser) or session-based (which means that they will be deleted once the browser is closed). Cookies have a limited storage capacity of 4KB.

In summary, cookies are used to store small amounts of data with a defined expiration date, while local storage and session storage provide larger storage capacity and do not have expiration dates (or have one tied to a session). Local storage is meant for long-term storage while session storage is useful for short-term storage within a session.

What Is Event Loop? How Does It Work?

ANS :

The event loop is a fundamental part of JavaScript's concurrency model, and it's responsible for managing the execution of asynchronous tasks in a single-threaded environment.The event loop has two main components: the call stack and the task queue. The call stack is a data structure that keeps track of the execution context of the currently executing code. When a function is called, its execution context is pushed onto the call stack, and when it returns, its context is popped off the stack.By using the event loop, JavaScript can handle multiple asynchronous tasks concurrently without blocking the execution of other code. It allows developers to write non-blocking code that is efficient and scalable. However, it's important to understand how the event loop works to avoid common pitfalls, such as blocking the event loop with long-running tasks or creating infinite loops.
